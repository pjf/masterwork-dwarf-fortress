Dwarf Therapist

A experimental build of Dwarf Therapist is included. I built the included version of Dwarf Therapist myself, it is untested on systems other than my own, please contact me (arclance) on the DF Forums if you have any problems with it. I have only tested this build on Debian (Testing, Unstable), xubuntu (11.10), and Fedora (14) please let us know if it works or not for you on other Distros.

This is a 32bit build, you may need some 32bit libraries to run it on a 64bit system. Because it is a 32 bit build it seems to be unable to detect the users QT theme (possibly GTK+ only) on 64bit systems.

To start Dwarf Therapist run the "DwarfTherapist" script in the "Dwarf Fortress" directory. Depending on your kernel version and build settings you may need to enable Dwarf Therapist to use ptrace on other processes.

You may get a error like this ptrace attach: Operation not permitted or this ptrace: Permission denied in your terminal when you run Dwarf Therapist. If your kernel does not have an extra LSM enabled or are running the Yama LSM (Ubuntu and others) you can enable ptrace by running this command as root in the "MasterworkDwarfFortress/Utilities/Dwarf_Therapist" directory:

sudo setcap cap_sys_ptrace=ep DwarfTherapist

This will allow the DwarfTheraist executable to ptrace other processes but will not turn off ptrace protection for the rest of your system. If your kernel is running SELinux (Fedora and others) you can enable ptrace by running this command as root:

setsebool deny_ptrace 0

This turns off all ptrace protection, ptrace protection will be reactivated if you reboot or run "setsebool deny_ptrace 1" as root.

Soundsense

Soundsense is included and can be found in "MasterworkDwarfFortress/Utilities/soundSense". Soundsense will work on Linux but may require more setup than in Windows. There are directions on running soundsense in Linux here. Please go to the soundsense thread for help if you have problems with soundsense. 

Masterwork Dwarf Fortress Settings.exe

The Masterwork Settings GUI (Masterwork Settings.exe) can work in Wine. It likely requries a recent version of Wine as well as a recent version of Mono being installed in your wine prefix. PhoebusTilesetAssembler.exe may also work in Wine with the same setup requirements. 

All other utilities

All other utilities (perfectworld, legendsviewer, and embark everywhere) have been removed until their functionality in Linux has been determined. If they are found to function acceptably in Linux they may be put back in the future.

Running Dwarf Fortress

Some parts of this mod depend on dfhack.

Poison coating and the steam engine both need dfhack to be active.

Dwarf Fortress is set to Windowed mode by default because Fullscreen is buggy on multiple monitor setups in Linux (uses all monitors without further user setup).

Linux users with multiple monitors can use the "export SDL_VIDEO_FULLSCREEN_DISPLAY=0" line in the "dfhack" startup script to set which monitor is used in fullscreen mode.

Truetype mode is disable due to extreme instability when used with some dfhack plugins (anything that changes menus). If you want to try it set [TRUETYPE:YES] in your "Dwarf Fortress/data/init/init.txt" file.

You may need to install some libraries that Dwarf Fortress depends on before it will run.

You may need to install the 32bit verison of some of these libraries to run Dwarf Fortress on a 64bit system. You can find instructions on how to do that here.

To start Dwarf Fortress with dfhack run the "dfhack" script in the "Dwarf Fortress" directory.

To start Dwarf Fortress without dfhack run the "df" script in the "Dwarf Fortress" directory.
